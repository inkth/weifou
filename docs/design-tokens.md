# 微否界面设计系统 · 人物编辑感与雾蓝紫 AI 气场

> 版本：2.0 · 2026-07-12
> 适用范围：微信小程序与 Flutter App。本文档是双端视觉令牌与界面原则的唯一真源。

## 1. 设计结论

微否全面采用 **人物编辑感 + 雾蓝紫 AI 环境光 + 局部成长色** 的视觉系统。

产品不是 AI 工具箱，也不是游戏化学习应用。界面首先服务于“一个真实的人”，AI 以安静、可信、持续在线的方式存在于人物周围。公开名片、分身聊天和分享场景必须让人物内容先于平台装饰被看见。

视觉配比：

- 80% 黑、白、冷灰中性色：承载人物、文字、结构与信任；
- 15% 雾蓝紫：表达分身气场、AI 状态、选中与关键操作；
- 5% 功能色：成功、提醒、风险与学习路径状态。

## 2. 产品视觉原则

### 2.1 人物先于 AI

- 真人头像、姓名、身份、经历和表达是公开页面的第一视觉层。
- AI 光晕只能衬托人物，不遮盖照片，不降低正文对比度。
- 吉祥物不进入公开名片；公开名片传播的是用户本人。

### 2.2 编辑感，而非工具面板感

- 使用明确的大标题、舒展留白、克制卡片和高对比排版。
- 不把所有内容都装入胶囊，不用颜色代替信息层级。
- 页面结构遵循“身份 → 状态 → 内容 → 行动”，避免入口宫格堆叠。

### 2.3 AI 是环境光

- 雾蓝紫用于头像光晕、生成、思考、记忆、选中和 AI 对话场景。
- 禁止全屏高饱和紫色、霓虹描边、装饰性发光文字和无意义渐变球。
- 同一人物的名片、聊天与海报延续一致的气场，可在蓝紫范围内轻微个性化。

### 2.4 清晰比“淡”更重要

- 浅灰只用于时间、占位和禁用状态；正常可操作内容不得呈现为禁用态。
- 正文与背景保持充分对比；渐变区域上的文字必须有稳定底或遮罩。
- 一屏只保留一个最高优先级的实底主按钮。

### 2.5 成长区允许有温度

“人类基本功计划”保留进度、路径色和完成反馈，但与主产品共享字体、卡片、圆角和阴影。颜色表达路径与状态，不表达玩具质感。

## 3. 核心颜色令牌

### 3.1 中性色

| 语义 | 小程序 / App | 值 | 用途 |
|---|---|---:|---|
| 主文字 | `--ink` / `ink` | `#17181C` | 标题、正文、墨黑行动 |
| 次级文字 | `--ink-2` / `ink2` | `#5E616D` | 副标题、辅助正文 |
| 三级文字 | `--sub` / `sub` | `#8D909B` | 说明、非活跃导航 |
| 四级文字 | `--faint` / `faint` | `#B5B7C0` | 时间、占位、禁用 |
| 发丝线 | `--line` / `line` | `#F0F1F5` | 内部分隔 |
| 边框 | `--border` / `border` | `#E7E8EE` | 卡片、输入、白色按钮 |
| 强边框 | `--border-strong` | `#D9DAE3` | 聚焦、浮层边界 |
| 中性填充 | `--fill` / `fill` | `#F3F4F7` | 标签、头像占位 |
| 按下填充 | `--fill-pressed` | `#ECEEF3` | 列表与白色控件按下 |

### 3.2 表面

| 语义 | 令牌 | 值 |
|---|---|---:|
| 页面背景 | `--bg` | `#F6F7FB` |
| 卡片表面 | `--surface` | `#FFFFFF` |
| 凹陷面 | `--surface-sunken` | `#F1F2F7` |
| 暖纸底 | `--paper` | `#FBFAF7` |

页面不使用纯白铺满所有层级。白卡在雾灰背景上自然形成结构，主要依靠边框和微弱阴影，而不是 3D 底唇。

### 3.3 品牌与 AI 气场

| 语义 | 令牌 | 值 | 用途 |
|---|---|---:|---|
| 品牌蓝紫 | `--accent` | `#7772C8` | AI 主行动、选中、焦点 |
| 深靛紫 | `--accent-strong` | `#5B569F` | 主按钮按下、强调文字 |
| 深色蓝紫 | `--accent-deep` | `#4F4A8B` | 浅紫底上的文字与图标 |
| 淡紫底 | `--accent-soft` | `#EFEDFA` | AI 标签、选中背景 |
| 淡紫底文字 | `--accent-ink` | `#47427D` | `accent-soft` 上的文字 |
| 实底文字 | `--on-accent` | `#FFFFFF` | 深蓝紫按钮文字 |
| 标题高亮 | `--hl` | `#E2DFF5` | 标题关键词的柔和底色 |
| 雾蓝 | `--mist-blue` | `#B9DDED` | 环境光、AI 渐变 |
| 雾紫 | `--mist-lilac` | `#C9C3EB` | 环境光、AI 渐变 |
| 蓝紫天空 | `--grad-sky` | `#E2DFF5 → #E2F0F6 → #F6F7FB` | onboarding、AI 空态 |

蓝紫色不是大面积装饰色。按钮、导航、光晕同时出现时，优先保留一个最有意义的蓝紫实体，其余使用中性色。

### 3.4 功能与成长色

| 语义 | 值 | 规则 |
|---|---:|---|
| 成功 | `#4F9D78` | 完成、在线、已连接 |
| 警示 | `#D39A4A` | 待处理、临期 |
| 危险 | `#D85C68` | 删除、失败、风险 |
| 成就金 | `#C7A45D` | 仅会员、掌握与里程碑 |
| 路径蓝 | `#73A8CF` | 学习路径辅助色 |
| 路径紫 | `#9A82C7` | 学习路径辅助色 |
| 路径杏 | `#D6A06F` | 学习路径辅助色 |

一屏辅助色不超过两种。成交、支付、认证区只使用中性色、蓝紫主色及必要状态色。

## 4. 空间、圆角与阴影

间距继续使用 4 基数：`8 / 16 / 24 / 32 / 40 / 56 / 72rpx`。页面横向留白为 `32rpx`，主区块间距为 `28rpx`。

圆角阶：

| 令牌 | 值 | 用途 |
|---|---:|---|
| `--r-sm` | `12rpx` | 标签、气泡细节 |
| `--r-md` | `20rpx` | 按钮、输入 |
| `--r-lg` | `26rpx` | 列表卡、气泡 |
| `--r-xl` | `32rpx` | 标准卡片 |
| `--r-2xl` | `40rpx` | Hero、公开名片 |
| `--r-3xl` | `48rpx` | 底部抽屉 |
| `--r-full` | `999rpx` | 头像、必要的状态胶囊 |

阴影规则：

- 发丝层：`0 2rpx 8rpx rgba(34, 32, 56, .04)`；
- 卡片层：`0 8rpx 28rpx rgba(34, 32, 56, .06)`；
- Hero 层：`0 18rpx 54rpx rgba(50, 46, 91, .10)`；
- 浮层：`0 24rpx 72rpx rgba(28, 27, 38, .16)`；
- 禁止用无模糊的垂直 `box-shadow: 0 X 0` 制造糖果底唇。

按压反馈使用 `scale(.985)`、轻微降低透明度或更换填充，不再整体下沉。

## 5. 字体与图标

- 使用系统字体栈：`-apple-system / PingFang SC / Helvetica Neue`。
- Hero `56rpx/800`、展示标题 `46rpx/800`、区段标题 `32rpx/750`、正文 `28rpx/400`。
- 大标题允许紧字距，正文保持自然字距和 `1.55–1.7` 行高。
- 功能图标统一使用 24 网格、2px 圆头线性图标；不得用 emoji 代替常规功能图标。
- emoji 只允许出现在庆祝、课程插画和用户生成内容中。

## 6. 组件规则

### 按钮

- 主按钮：深蓝紫或墨黑实底、白字、中等圆角、无 3D 底唇；
- AI 主行动使用深蓝紫；人物关系与通用确认可使用墨黑；
- 次按钮：白底、浅灰边、深色文字；
- 文本按钮：不额外套胶囊；
- 禁用态才降低到 42% 不透明度。

### 卡片

- 标准卡片白底、浅灰边、微弱阴影；
- Hero 卡可增加雾蓝紫环境光，但内容层必须保持清晰；
- 不为每个菜单行单独套卡，也不为装饰目的叠卡。

### 标签

- 标签只表达可扫描的身份、属性或状态；
- 默认灰底，AI/选中标签使用淡紫底；
- 标签使用小圆角或轻胶囊，避免所有按钮和筛选器都变成胶囊。

### 输入与聊天

- 输入区白底、灰边，聚焦时变蓝紫边；
- AI 气泡白底或淡紫底，用户气泡深蓝紫底；
- 聊天背景可使用极淡雾蓝紫渐变，气泡必须保持稳定对比；
- 始终明确展示“某人的 AI 分身”以及必要的隐私说明。

### 底部导航

- 使用贴合页面的全宽平面底栏，不使用悬浮胶囊、外框或大阴影；
- 未选中图标与文字使用三级灰，选中使用深靛紫；
- 选中态依靠图标实心/线性变化与颜色表达，不增加大面积选中底块；
- 公开名片、聊天和创建分身等沉浸页面隐藏底栏。

## 7. 页面级应用

| 页面 | 设计重点 |
|---|---|
| 首页 | “我的 AI 身份”是第一资产；展示分身状态、访问与待处理关系，不做工具宫格 |
| 公开名片 | 人物照片/头像最大；身份、人格、可信信息和开聊行动构成主路径 |
| 分身聊天 | 延续人物气场；清晰区分 AI、本人和访客；输入与行动稳定可读 |
| 消息/连接 | 高密度但安静的列表；状态依靠粗细、圆点和少量蓝紫 |
| 我的 | 分身与分享是核心资产；设置项回归清晰列表，不呈现为禁用态 |
| 创建分身 | 单任务分步表单；一屏一个问题，明确进度和生成预期 |
| 修炼 | 保留路径色、进度和完成反馈，移除糖果底唇和多色轰炸 |
| 会员/支付 | 白底、深色排版、蓝紫主行动；不靠大面积金色制造高级感 |

### 课程学习页

- 采用“单张任务卡原地推进”，完整消息只进入学习记录，不在主界面持续堆叠；
- 固定顺序为：情境/任务 → 一次选择 → 即时解释 → 下一步；
- 上方使用克制的 2D/2.5D 数字人陪学舞台，只表达就绪、倾听与讲解，不使用对峙、战斗和角色走路等游戏表演；
- 数字人待机时保留低强度呼吸，以随机节奏偶发眨眼、看向题卡和姿态调整；45 秒无操作只出现一次安静文字提示，不自动说话、不震动、不反复催促；
- 用户选择后先用约 300ms 的注视/点头表达“已听见”，再进入思考；讲解时按文本和标点驱动闭口、小开口、横向开口与圆口嘴型，状态切换保持平滑；
- 数字人语音默认由用户主动播放，只朗读当前解释的短摘要，必须同步保留可读文字；
- 普通答对在当前界面使用对勾、数字人点头、环境光与轻震的微庆祝，不弹窗、不切屏；答错只给温和提示，不使用红叉和失败音效；
- 学习进度、能力面板、历史和复习入口默认收起，不在主学习界面呈现地图；
- 学习中只显示紧凑课程头、数字人舞台、当前任务卡和 2–4 个等权选项；
- 每次普通答对不触发全屏庆祝，彩带只用于掌握、通关和连续学习里程碑；
- 英语课保留朗读等学科专属交互，其他路径共享结构但不强行共享题型。

## 8. 品牌资产

盆栽 Logo 保留原有不对称树冠、冠盆间隙和几何特征，并统一输出蓝紫版本：

- 深底：`#232238`；
- 树冠/主体：`#7772C8`；
- 陶盆/次级高光：`#B9DDED`；
- 单色场景使用 `#4F4A8B`，深底场景整体反白。

SVG 与 PNG 必须由 `assets/brand/logo.svg` 和 `scripts/gen-logo.mjs` 同步生成，禁止手工修改单个尺寸。

## 9. 双端维护

1. 修改颜色或空间令牌时，先更新本文档；
2. 同步小程序 `app.wxss` 与 Flutter `AppTheme`；
3. 自定义 tab bar 因样式隔离，需要同步令牌回退值；
4. 页面内禁止新增品牌绿和糖果底唇；
5. 新增渐变必须说明其业务语义，不能仅用于装饰；
6. 每次改版检查公开名片、聊天、首页、修炼和会员五类代表页面。

## 10. 验收标准

- 全产品不存在品牌绿作为 UI 强调色；
- 正常功能不使用类似禁用态的浅灰文字；
- 不存在糖果式无模糊 3D 底唇；
- 公开名片与聊天优先展示人物而非平台装饰；
- 蓝紫实底元素一屏通常不超过一个主行动；
- AI 渐变不影响正文、按钮和消息气泡的对比度；
- 小程序与 App 的核心颜色、按钮、圆角和阴影语义一致。
